Usync

简介​

Usync 数据同步组件，是基于开源 Datax 进行二次开发的一款异构数据源离线同步工具，通过简单的 json 文件配置可以实现在任意的数据处理系统之间的数据交换。实现包括 MySQL、SQL Server、Oracle、HDFS、Hive、Ftp、Doris、kingbasees、Elasticsearch 等各种异构数据源之间高效的数据同步功能。Usync 不仅支持对结构化数据的同步，还支持图片类数据的传输。同时提供了数据转换处理、限速和运行指标监控等强大功能。为用户快速搭建高效稳定数据同步平台，降低使用的门槛。

特性​

  • Usync 本身作为数据同步框架，将不同数据源的同步抽象为从源头数据源读取数据的 Reader 插件，以及向目标端写入数据的 Writer插件，理论上Usync 框架可以支持任意数据源类型的数据同步工作。同时 Usync 插件体系作为一套生态系统, 每接入一套新数据源该新加入的数据源即可实现和现有的数据源互通。
  • 支持 FTP 、HDFS 、HIVE 、TXTFile 数据源底层通过二进制流式传输。
  • 支持Hive的数据读取和Hive的数据写入,支持Doris的数据高效稳定写入。
  • 提供数据安全能力,数据脱敏，数据加解密等。
  • 架构调整，减小项目臃肿，加快程序编译。

技术框架​

  • Linux
  • JDK(1.8以上，推荐1.8)
  • Python(2或3都可以)
  • Apache Maven 3.x (Compile Usync )

快速搭建​

工具部署​

  • 直接下载 Usync 工具包

    下载后解压至本地某个目录，进入bin目录，即可运行同步作业：

    $ cd  {YOUR_USYNC_HOME}/bin
    $ python usync.py {YOUR_JOB.json}

    自检脚本： python {YOUR_USYNC_HOME}/bin/usync.py {YOUR_USYNC_HOME}/job/job.json

核心优势

可靠的数据质量监控​

  • 完美解决数据传输个别类型失真问题

    Usync旧版对于部分数据类型(比如时间戳)传输一直存在毫秒阶段等数据失真情况，新版本Usync已经做到支持所有的强数据类型，每一种插件都有自己的数据类型转换策略，让数据可以完整无损的传输到目的端。

  • 提供作业全链路的流量、数据量运行时监控

    Usync运行过程中可以将作业本身状态、数据流量、数据速度、执行进度等信息进行全面的展示，让用户可以实时了解作业状态。并可在作业执行过程中智能判断源端和目的端的速度对比情况，给予用户更多性能排查信息。

  • 提供脏数据探测

    在大量数据的传输过程中，必定会由于各种原因导致很多数据传输报错(比如类型转换错误)，这种数据Usync认为就是脏数据。Usync目前可以实现脏数据精确过滤、识别、采集、展示，为用户提供多种的脏数据处理模式，让用户准确把控数据质量大关！

丰富的数据转换功能​

Usync作为一个服务于大数据的ETL工具，除了提供数据快照搬迁功能之外，还提供了丰富数据转换的功能，让数据在传输过程中可以轻松完成数据脱敏，补全，过滤等数据转换功能，另外还提供了自动groovy函数，让用户自定义转换函数。详情请看Usync的transformer转换.md

精准的速度控制​

还在为同步过程对在线存储压力影响而担心吗？新版本Usync提供了包括通道(并发)、记录流、字节流三种流控模式，可以随意控制你的作业速度，让你的作业在库可以承受的范围内达到最佳的同步速度。

"speed": {
   "channel": 5,
   "byte": 1048576,
   "record": 10000
}

二进制传输​

Usync 中的Ftp只支持text或者csv文件进行传输，无法输出图片。我们对Usync文件类数据添加支持二进制模式传输。实现了图片传输。

除此之外，二进制传输模式能保证各种格式的文件准确传输，目前支持二进制模式传输的组件主要是：hdfs、ftp、txtFile。

{
  "job": {
    "setting": {
      "transport": {
        "type": "stream"
      },
      ...
    },
    ...
  }
}

强劲的同步性能​

Usync 每一种读插件都有一种或多种切分策略，都能将作业合理切分成多个Task并行执行，单机多线程执行模型可以让Usync 速度随并发成线性增长。在源端和目的端性能都足够的情况下，单个作业一定可以打满网卡。另外，社区团队对所有的已经接入的插件都做了极致的性能优化，并且做了完整的性能测试。

健壮的容错机制​

Usync 作业是极易受外部因素的干扰，网络闪断、数据源不稳定等因素很容易让同步到一半的作业报错停止。因此稳定性是Usync 的基本要求，在新版Usync的设计中，重点完善了框架和插件的稳定性。目前Usync可以做到线程级别、进程级别(暂时未开放)、作业级别多层次局部/全局的重试，保证用户的作业稳定运行。

  • 线程内部重试

    Usync的核心插件都经过团队的全盘review，不同的网络交互方式都有不同的重试策略。

  • 线程级别重试

    目前Usync已经可以实现TaskFailover，针对于中间失败的Task，Usync框架可以做到整个Task级别的重新调度。

极简的使用体验​

  • 易用

    下载即可用，支持linux和windows，只需要短短几步骤就可以完成数据的传输。

  • 详细

    Usync在运行日志中打印了大量信息，其中包括传输速度，Reader、Writer性能，进程CPU，JVM和GC情况等等。

    • 传输过程中打印传输速度、进度等

    • 传输过程中会打印进程相关的CPU、JVM等

  • 在任务结束之后，打印总体运行情况
